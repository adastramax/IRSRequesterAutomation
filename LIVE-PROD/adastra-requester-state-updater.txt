Context for the IRS PIN Creation Agent

This project lives at E:\ad-astra\039 Requester Migration State Site Script and is complete for Customer 1222. It automated the correction of wrong IRS requester address and state fields in the Ad Astra Connect platform. The IRS PIN creation project may reuse several pieces of this work.

The Core Data Relationship
IRS requesters in Connect each have a 9-digit PIN stored as pinCodeString. The first 4 digits of that PIN are the TEID (TE ID), which maps to an IRS site name and state in the authoritative SharePoint workbook:

pinCodeString = "674391786"
TEID = "6743"  → IRS W&I FA Area 4, Group 411, San Diego / CA
Connect API Endpoints (all confirmed working)
1. Auth

Body (JSON): {"email": "aiagentdjango@adastra-inc.com", "password": "Aiagent123@"}
Response: {"token": "...", "isAuthenticated": true, ...}
2. Bulk requester export (full population)

POST /api/accounts/exports/filter/CONSUMER/0/?page=1&items_per_page=99999
Body (JSON):
{
  "joinDate": null,
  "loginDate": null,
  "customers": [1222],
  "status": [],
  "roles": []
}
Customer 1222 = US GSA IRS TS Field Assistance (FA)
3. SEID (single requester) search

POST /api/accounts/members/filter/CONSUMER/0/?page=1&items_per_page=50&search=<SEID>
Body: same JSON filter payload
SEID is stored as firstName in Connect
4. Full requester detail

GET /api/Accounts/GetAccountDetailByID/{code}
Response: {"data": {...full profile...}, "payload": null}
The actual detail is under data, not top-level
Returns pinCodeString, pinCode, address, state, lockoutEnabled, serviceTypes, and all other profile fields needed for update
5. Update requester

POST /api/accounts/Update
Content-Type: application/x-www-form-urlencoded   ← CRITICAL: NOT JSON
Response: {"status": "S", "text": "Successfully updated", "result": "<code>"}
Sending JSON returns HTTP 400 even with all required fields present
Payload must be sent as data= (list of tuples / form-encoded)
serviceTypes arrays use ASP.NET Core syntax: serviceTypes[0].value, serviceTypes[0].label
You must send the full account payload — fetch detail first, then overwrite only the fields you want changed
